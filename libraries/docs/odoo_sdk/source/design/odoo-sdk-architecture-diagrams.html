The SDK is the boundary between consumer Python code and Odoo's XML-RPC interface. Extensions should attach at the SDK layer, not directly to transport internals.

The recommended architecture adds an environment, recordset abstraction, metadata cache, field adaptation, and x2many boundaries while preserving a simple top-level client facade. Plugin hooks remain a Phase C seam.

Metadata lookup, serialization, transport, and error mapping each get a dedicated boundary so ORM-like features can evolve without leaking XML-RPC details.

The roadmap is staged so the missing abstractions land first, field semantics second, and richer extensibility only after the local workflow and core contracts are stable.
